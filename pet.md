# ペット防災シミュレーションゲーム シナリオ仕様書・開発指示書

## 1. 基本設定
* **主人公:** 女性(20代後半)
* **住居:** マンション8階住み
* **ペット:** 犬 (マルチーズ4歳)
* **季節・天候:** 夏、晴れ
* **ゲーム構成:** メインキャラ中心にシーンを構成し、要所で問題を出題する。正解すると次のシーンへ進み、不正解の場合はシーンが変わらず不正解の表示が出る。

---

## 2. シナリオ進行と選択肢 (正規ストーリー)

### シーン1: 地震発生
* **状況:** 主人公が料理(肉じゃが)をしている最中に地震が発生し、犬が鳴き出す。
* **【問1】** ガタガタガタ・・ え?!地震?! かくれなきゃ!!
    * **選択肢①:** 火を消して犬と一緒にテーブルの下にもぐる (正解: 震度5弱以上で自動的に消えるため、無理に火に近づかない。自分の身とペットを守るのが大事)。
    * **選択肢②:** 火を消さずに〃。
    * **選択肢③:** 料理をつづけて犬もご飯を食べる (不正解)。

### シーン2: 避難準備
* **状況:** 揺れが収まり、ニュースで避難勧告を確認。外から煙が入ってくるため火事の危険がある。
* **【問2】** 犬どうしよう……。
    * **選択肢①:** 一旦、家に置いていく (不正解: 戻って来られるとは限らない)。
    * **選択肢②:** リードを付けて逃げる (不正解: 避難所に入れないかも)。
    * **選択肢③:** リードを付けてクレートに入れて一緒に連れて避難しよう (正解: 他の動物や人とトラブルもあるかもしれないため)。

### シーン3: 移動中の余震
* **状況:** エレベーターが止まっているため階段で外へ出る。その直後、また地震(余震)が発生する。
* **【問3】** はっ!!! また地震だ!!!どうしよう!
    * **選択肢①:** 慌てて建物の中に入る (不正解: 建物の崩壊に巻き込まれる危険がある)。
    * **選択肢②:** ガラスの破片などが落ちてきそうな場所から離れて、かがんでペットと頭を守る (正解)。
    * **選択肢③:** 走って避難所に向かう (不正解: 揺れが収まるまでは動いたら危ない)。

### シーン4: 避難所到着
* **状況:** 揺れが収まり避難所へ向かう。受付で犬連れ専用の場所へ案内され、教室に到着する。
* **【問4】** 避難所が指定した場所で
    * **選択肢①:** クレートに入れておく (正解: 避難所での基本原則)。
    * **選択肢②:** クレートから出して自由にさせる (不正解: 逃走の恐れあり)。
    * **選択肢③:** リードでつないでおく (不正解: 周りの人のことも考えなければならない)。

## 3. バッドエンド (避難所に受け入れ不可パターン)
* **状況:** 避難所で「動物のうけ入れおこなってません」と断られる。
* **結果:** とりあえず家に帰るが、家が倒壊しており「なんてこった」となる。
* **こうならないための事前対策:**
    * 事前に近所の避難所がペット受け入れしているか複数確認し、一度その経路をペットを連れて歩いてみる。
    * 親せき、友人、ペットホテル、動物病院など、一時的に預けてもらえる人がいるか相談・確認しておく。
    * 自治体がペット同行避難について、どのような方針を示しているか確認する。
    * ペットの大きさや種類による受け入れ状況を確認しておく。

## ⚙️ AIエージェント（Antigravity等）へのシステム実装指示

本プロジェクトを実装するにあたり、以下の仕様とアプローチを厳守してコーディングを行ってください。

### 1. アプリケーション設計 (Web / Mobile 両対応)
* **推奨技術スタック:** React (または Next.js) + Tailwind CSS
* **レスポンシブ設計:** スマートフォンアプリとしての利用と、Webブラウザでの利用の両方を想定しています。モバイルファースト（Mobile-first）のレスポンシブWebアプリとしてUIを構築してください。
* ネイティブアプリ（iOS/Android）特有の複雑なビルドは避け、ブラウザで快適に動作する仕様（PWA化も視野に）としてください。

### 2. 画像（イラスト）とアセットの取り扱い方針
* **外部画像APIの使用禁止:** Unsplashなどの画像APIを使用した動的な画像取得は**絶対に行わないでください**。防災の具体的なシチュエーション（例：犬と一緒にテーブルの下に隠れる）に合致する画像はAPIでは取得できず、ゲームの没入感を損なうためです。
* **プレースホルダーの活用:** 開発の初期段階（MVP）では、すべてのイラスト配置予定箇所に「色付きのプレースホルダー（例: 灰色のボックス）」を配置してレイアウトを組んでください。
* **JSON駆動型のデータ構造:** 上記のシナリオデータと選択肢ロジックはJSON形式で管理し、各シーンのデータ内に画像パス（例: `"image": "/assets/scene_01.png"`）を定義する構造にしてください。これにより、後日チームメンバーが `assets` フォルダに画像をアップロード・差し替えるだけで、コードを修正せずに画像を反映できる設計にしてください。
